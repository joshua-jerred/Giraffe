bool PSK::encodeTextData ( std::string  message)

Encode a string of text data into PSK audio data and save to wave file.

Adds preamble and postamble to the audio data and if a callsign was specified, adds morse code callsign to the beginning and end of the audio.

This method will use varicode with two 0's between each character. If in QPSK mode, it will use the convolutional code.

Parameters
message
Returns
true - Success, you can find the wav file at the file path specified
false - Failure, check the console for more information

int  length

Encode raw data into PSK audio.

Adds preamble and postamble to the audio, and callsign if specified. Does not use varicode or convolutional code! This should only be used if you are manually decoding. You will be forced to use a morse callsign if you use this method. You can of course change this if you want to, but please keep in mind that if you use your own encoding people will not be able to identify your station!

Adds a bit to the bit stream.

see format of 'bit stream' in header file. This function assumes that you're adding bits in order (left to right). So if you want to add 1 you must add 0x80 with a 'num_bits' of 1. All numbers past 'num_bits' must be 0.

◆ bit_stream_

std::vector<uint32_t> PSK::bit_stream_
private

Bit stream is an array of 32 bit integers. The bits are stored in a 32 bit integer in the following way:

Characters: a a b b c c

varicode with 2x 0 bit padding added to the end of each character: '1011' '00' '1011' '00' '1011111' '00' '1011111' '00' '101111' '00' '101111' '00'

32 bit array: [0](10110010110010111110010111110010) {a, a, b, b, half of c } [1](11110010111100000000000000000000) {other half of c, 0 bits..}

Although this may not be the best way to store the bit stream, it makes it easier to understand.
