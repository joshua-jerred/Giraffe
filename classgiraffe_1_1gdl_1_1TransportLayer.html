Detailed Description

The Transport Layer - Layer 3.

The transport layer is responsible for sending messages from the application layer (DataLink) to the network layer (NetworkLayer) and receiving messages from the network layer and passing them to the application layer. The transport layer will look at the messages that is must send and will route them to the correct portion of the network layer.

Along with all of this, the transport layer is responsible for maintaining the connection between the local and remote nodes.

◆ last_acked_message_id_

uint32_t giraffe::gdl::TransportLayer::last_acked_message_id_ = 0xffffffff
private

The last message id that was acked by the local node.

This is used to determine if the message should be pushed to the DataLink layer or not. If the message id of the next received message does not match this, it will be pushed up. If the message id of the received message matches this, it will not be pushed up but it will still be acked.
