◆ stop()

void modules::Module::stop ( )
inherited

Stops the module by setting the stop flag to true and joining the runner thread.

This function will stop the module by setting the stop flag to true and joining the runner thread. It also sets the status to STOPPED. If a module is in the sleep state, it will only stop after the next loop. This means that the sleep interval plus the time it takes to run the loop and shutdown will be the total time it takes to stop the module.

References node::STOPPED, and node::STOPPING.

This function is responsible for parsing a request and routing it to the right place.

The function handles incoming requests from a TCP socket server by parsing them as JSON strings and checking their type and category before sending the appropriate response or an error packet if they are not implemented or malformed.

Parameters
clientThe client parameter is a reference to a sock::TcpSocketServer object, which represents the client that sent the request to the server.
requestThe request parameter is a reference to a std::string object that contains the json message received from the client.
