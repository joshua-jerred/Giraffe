◆ ax25::Timers

struct ax25::Timers

Timers for the state machines. See 6.7 of the AX.25 2.2 specification.

Collaboration diagram for ax25::Timers:
Class Members
Timer t1 Acknowledgement Timer (T1)

Ensures that the TNC does not wait indefinitely for a response to a frame it sends.

Timer t2 Response Delay Timer (T2)

Optionally implemented by a TNC to specify a maximum amount of time delay to be introduces between the time an I frame is received and the time the resulting response is sent.

Timer t3 Inactive Link Timer (T3)

Maintains the link integrity whenever T1 is not running.

Timer t100 Repeater Hand Timer (T100/AXHANG)

Tracks the amount of time an audio repeater will keep it's transmitter keyed after it stops receiving a signal.

Timer t101 Priority Window Timer (T101/PRIACK)

Prevents stations from transmitting non-priority frames during the first available transmission time slow. The first slow is reserved for priority frames (acks, digipeat)

Timer t102 Slot Timer Timer (T102/p-persistence)

Randomly delays stations before they transmit a frame to help prevent collisions.

Timer t103 Transmitter Startup Timer (T103/TXDELAY)

Allows time for a transmitter to stabilize before transmitting.

Timer t104 Repeater Startup Timer (T104/AXDELAY)

Allows time for a repeater to start up before transmitting.

Timer t105 Remote Receiver Sync Timer (T105)

Introduces additional delay after TXDELAY, if needed, to allow a remote receiver to sync up before transmitting frames.

Timer t106 Ten Minute Transmission Timer (T106)

Ensures that a station is not keyed for more than 10 minutes.

Timer t107 Anti-Hogging Timer (T107)

Prevents a station from monopolizing the channel.

Timer t108 Receiver Startup Timer (T108)

Allows time for a receiver to start up after transmitting to ensure that the frequency is clear before transmitting again.

Timer tr210 Next Segment Timer (TR210)

Ensures that the reassembler doesn't wait forever for the next segment.
